4. Make Your First Map¶

The very first command in the beginning is definitely pscoast, which is also the easiest command for mapping. In this chapter, we’re going to see how to make a map with coastlines using pscoast. The other topics in this chapter are the style of GMT input and how to set up these input parameters.

4.1. Goal¶

Make a map of Archipelago Sea, as the figure here shows. Located in Northern Europe, Archipelago Sea is actually part of the Baltic Sea. As its name suggests, the sea are actually occupied by numerous islands in any size. Although Most islands are small, Archipelago Sea possibly contains the most island in the world. The largest island in the archipelago is Åland, an autonomous region of Finland. Such a place makes itself ideal to show how pscoast can fine tune your coasline map.

4.3. Procedure¶

The first step of plotting a map is to determine where to be plotted. Even though we know we are going to plot Archipelago Sea, we still can’t start without a precise map extent. The most convenient way to retrieve map extent in Lat/Long representation is to go to Google Maps and search for Archipelago Sea. When the map jumps to the region of our interest, right-click on “What’s here?”, and the name of the region and its Lat/Long would appear in the bottom.

Note how longitude and latitude are represented here: there are 2 numbers in the pop-up window. The first one 59.865253 is latitude; positive and negative value are N and S respectively. The second one 19.435697 is longitude; positive and negative value are E and W respectively. Also, it does not use the “degrees, minutes, secounds” convention; instead, it converts all minutes and seconds and gives you a real number in degrees. It’s much easier to use this format in math calculation, and thus GMT adopts it as the default representation for geographical coordinates.

After taking a look, we are going to select longitude from 19.42 to 22.95 degrees, and latitude from 59.71 to 60.56 degrees for mapping. Also, feel free to try whatever region if you would like! Now, open Terminal and enter the following prompt:

$ pscoast

You would see a few pages that show all the syntax and options of pscoast. If you’ve already been super familiar with these options and just want to make sure for the usage, you can also enter command like the following line, ant it will only show the syntax on screen.

$ pscoast -

Almost all GMT commands, including pscoast have a syntax like this:

$ command (input_file) [option1 option2 option3...]

Every string starting with a minus sign (-) is interpreted as an option, or more technically speaking, a flag or an argument. If a string doesn’t start with a minus sign, it could only be the input file(s). Every GMT command usually has a required option or two; if other options are omitted, defaults settings are used. The order between options and input files don’t matter at all. That means, for example, it is ok to enter all options and put the name of input file in the end of line.

pscoast has got a little special: an input file is not required! Given by that, how does the program know where the coastline is? The answer is simple: there is a database called GSHHG along with the standard installation of GMT. The database contains a lot of basic information of the world, like coastline, country borders, rivers, etc. What pscoast does is to load data from GSHHS and plot them on your map. Thus, pscoast is not only capable for drawing coastline but is also used for drawing water bodies and countries. When using pscoast, we only need to focus on setting up options! The first required flag is -R (Region), which is used to specify extent (covered area) of a map. We can just put all the parameters directly after -R, without any spaces. Normally, there are two ways to specify extent using -R flag:

-R19.42/22.95/59.71/60.56
-R19.42/59.71/22.95/60.56r

The first format is W_Border/E_Border/S_Border/N_Border; that is, we specify the region by x coordinate (longitude) and y coordinate (latitude). The second format needs the coordiates of the bottom-left (BL) corner and the top-right (TR) corner, so it becomes BL_long/BL_lat/TR_long/TR_lat. Don’t forget to add an r if you are using the second way. No matter whichever way you use, it would give you the same result. Here we use the first format and enter the following line in the command prompt:

$ pscoast -R19.42/22.95/59.71/60.56
pscoast: Syntax error: Must specify a map projection with the -J option
pscoast: Syntax error: Must specify at least one of -C, -G, -S, -I, -N, -Q and -W

Oops, an error jumps. The issue here is that we should specify other required arguments as well. Except for map extent, a map projection is also very important for a map. in GMT, -J flag is used for assigning map projection. There are actually various projections at your service, but here we are going to pick a common one for drawing a regional map: Mercator projection. Enter the following command:

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i

The M after the -J is the code of Mercator projection, and 6i means that the width of the map is 6 inches. If you don’t know how long 6 inches are, just change it into -JM15c. This means the width of the map is 15 centimeters, which is roughly 6 inches wide.

Attention

-J and -R are actually related to each other. When the map projection changes, the way to set map extent and size also changes a little bit. Please see GMT official documentation or later chapter. For the usage of different map projections, please see later chapter or look for it in any GIS or geodesy textbooks. In addition, the default geodetic datum of GMT is WGS84 reference ellipsoid. If you would like to change it, plase see later chapter as well.

Now we still have an error left, which is about the “drawing style”. GMT asks us to specify at least one drawing style. Firstly, we use -W for the coastline pen attribute. A pen attribute contains 3 values: thickness, color, and style. The syntax is

-Wlinewidth,linecolor,linestyle

If any is omitted, the default is used. Here we try to specify a black pen with a thickness at 0.1 points (1 point = 1/72 inches = 0.03527 cm):

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black

0.1p just means 0.1 points, and black tells GMT to use black pen. Obviously we didn’t specify line style here, so GMT will use a soild line. Note that here we used the pre-defined color name in GMT, and all of these names are listed here.

Note

We will learn the advanced setting for pen attribute, like setting line style or using RGB custom color, in the chapter of “Pen and Painting.”

Now we have set 3 arguments up and pscoast should be able to run smoothly. Type the following command, and if you see this:

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black
%!PS-Adobe-3.0
%%BoundingBox: 0 0 595 842
%%HiResBoundingBox: 0 0 595 842
%%Title: GMT v5.1.1 (r12968) [64-bit] Document from pscoast
%%Creator: GMT5
..... #(it's pretty long!)
end
%%EOF

This means the map has generated successfully! Well, strictly speaking, GMT does not generate a map image but many text lines written in PostScript language. We have to save the codes as a text file, and than open it using software that can interpret this language. It’s east to save the output as a file using a common redirecting command in UNIX systems. It is

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black > archi_sea.ps

The last part > archi_sea.ps redirects the text output into archi_sea.ps. .ps is a common file extension for PostScript files. Now we are ready to open it using any PostScript viewer.

This figure has been rotated so the north is up, for increasing clarity. The actual map output should be positioned like the next figure. Nevertheless, This figure looks pretty bad since it doesn’t look like the what the targeted area should be. This is because we only used the minimum required arguments. For example, the coastline data used here is too crude to represent the actual shape of islands. Now we are going to add a few more flags to make it better match the real world. First, let us add color for land and sea. We will need -G (Ground) and -S (Sea) flags respectively.

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black -Gdarkseagreen2 -Scornflowerblue > archi_sea.ps

Like black, darkseagreen2 and cornflowerblue are also pre-defined color name in GMT. Now the command gets a bit longer, which is quite normal when using GMT, especially when you want to change the defaults and make a pretty map. It’s not a problem when a long command are entered at the prompt line because the terminal window would automatically cut a long command into a new line. However, if later you are writing a GMT script and want to make you script neat, you can use the UNIX line-break sign \ to continue the command at a new line. The following example would give the same result as the previous one:

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black -Gdarkseagreen2 \
          -Scornflowerblue > archi_sea.ps

In this tutorial, we also use \ to break a long command for clarity. If you would like to finish your command in one line, just ignore this line-break sign.

Next, we are going to adjust the coastline resolution. This is because some islands in this region are so small that they won’t show on a map without a finest resolution. pscoast use -D flag to control the coastline resolution. The accepted parameters are f (full), h (high), i (intermediate), l (low), and c (crude). The default value is l. Here we use the highest level of resolution, that is, -Df:

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black -Gdarkseagreen2 \
          -Scornflowerblue -Df > archi_sea.ps

The following figure is the output .ps file.

Here we can see that the map is totally changed by only adjusting coastline resolution and colors for land and sea.

Maybe you’ve noticed that there is a always large white space in the output ps file. This is because GMT is not “generating an image of map” but “printing a map on a fixed paper format”. If we stick to the default settings, GMT would use landscape-oriented A4 paper. Therefore, our 15-cm-wide map does not totally occupy the paper and leaves a large portion of empty white space. Here we are not going to adjust the paper size, but will address the paper orientation because many PostScript viewers use portrait A4 as default (Microsoft Word does this, too). If we open a landscape-oriented A4 map using the default setting, the map would be rotated and thus the north is not at the top anymore, which is not a conventional map setting. To be more convenient, we can add -P flag, which tells GMT the map should be drawn on a protrait-oriented A4 paper.

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black -Gdarkseagreen2 \
          -Scornflowerblue -Df -P > archi_sea.ps

Tip

To change paper size, you need to learn the settings in gmtdefaults. Likewise, you can also change the settings in gmtdefaults directly and then there will be no need to add -P. Please see later chapter for setting up paper format in detail.

Now the main map is almost finished, and it’s time to add a map frame. GMT uses -B to tweak map frame and associated elements, like grids, ticks, annotations and title. They are collectively called the Basemap settings. -B is one of the most complex arguments in GMT since it’s got tons of styles for your choice. Here we are going to introduce 2 simplest combinations:

-Ba(annotation_spacing)f(tick_spacing)g(grid_spacing)
-B(four_borders)+t(title)

The first combination gives the spacing number to annotation (tick mark), tick, and grid. It used the same unit as specified in -R. For example, -Ba1f1g1 means annotations, ticks, and grids are plotted at every degree; -Ba2f1 means that “annotation is labeled every other degree, tick is plotted every degree, and don’t add any grid.” The second combination is for four sides of a map, and it uses W, S, E, N for West, South, East, and North. If it’s a capital letter, GMT would put everything (frame and annotations) at this side; if it’s a lowercase letter, only frame would be added to this side; if you don’t add a letter, nothing will be plotted. For example, -Bwsne is to put a map frame without any annotation (ignoring the -Ba setting in the first combination); -BWSe is to put a frame with annotation at west and south side, but only frame itself at east side (and no frame at the north - the map would look broken!) It’s also ok to add +t at the second combination, which a map title followed by the +t flag. Two combinations can be both used in the same command. For instance, we can design our map like this:

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black -Gdarkseagreen2 \
          -Scornflowerblue -Df -P -Ba2f0.5g1 -BWSne+t"Archipelago Sea" \
          > archi_sea.ps

In this case, a map frame us added to all four edges, but annotations are only marked at the west side and the south side. Lat/long annotations are at every 2 degrees, ticks (the black and white bars in the map frame) are 0.5 degrees wide, and grid line is put at every degree. The map is also entitled as Archipelago Sea, like the following figure shows:

Tip

Actually -B is a shortened version of another GMT command called psbasemap. We can use psbasemap to plot many more complex frame settings independently, and then filling the map content into the frame is just simple. For advanced -B options like custom tick marks, adjusting single axis, primary and secondary grid, and special characters, please see later chapter.

You would probably feel that ther are too many islands in Archipelago Sea and they easily distract readers, especially if we want to label other information on the map. To simplify our map and make readers focus on key information, pscoast provides -A flag to let us specify a minimum area of an island, in square kilometers. Once it’s set, pscoast would ignore any island that is smaller than the given area. Here we are going to make islands that are smaller than 0.1 square kilometers disappear:

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black -Gdarkseagreen2 \
          -Scornflowerblue -Df -P -Ba2f0.5g1 -BWSne+t"Archipelago Sea" \
          -A0.1 > archi_sea.ps

With fewer islands, the map looks cleaner but still keeps the sense of its name, “Archipelago Sea”.

Now our map is close to finish. Lastly, let us use -U flag to add a GMT mark and time stamp at the bottom right corner. The usage of -U is similar to -P - it doesn’t need any parameters.

4.4. Script¶

The final command for our goal is:

$ pscoast -R19.42/22.95/59.71/60.56 -JM6i -W0.1p,black -Gdarkseagreen2 \
          -Scornflowerblue -Df -P -Ba2f0.5g1 -BWSne+t"Archipelago Sea" \
          -A0.1 -U > archi_sea.ps

Note

“Plot the coastline map - map extent is set to (19.42E - 22.95E, 59.71N - 60.56N) in Mercator projection and 6 inches wide. Coastlines are black and 0.1 point wide. Land area is filled with the color darkseagreen2, and water area is filled with cornflowerblue. We use the coastline data at full resolution. The map is plotted in portrait mode, with a border frame and annotation marks in the left and bottom. Annotation spacing is 2 degrees, tick spacing is 0.5 degrees, and grid spacing is 1 degree. The map is entitled Archipelago Sea and saved as archi_sea.ps.”
